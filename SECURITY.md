# Security

Microsoft takes the security of our software products and services seriously, which includes all source code repositories managed through our GitHub organizations, including [this one](https://github.com/JiriHybl/get-DOprerequisites).

If you believe you have found a security vulnerability in this repository, please **do not** report it through public GitHub issues.

---

## Reporting a Vulnerability

Please report security vulnerabilities through one of the following channels:

- **Microsoft Security Response Center (MSRC):** [https://msrc.microsoft.com/create-report](https://msrc.microsoft.com/create-report)
- **Email:** [secure@microsoft.com](mailto:secure@microsoft.com)

You should receive a response within 24 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

---

## What to Include in Your Report

To help us understand the nature and scope of the possible issue, please include as much of the following information as possible:

- **Type of issue** (e.g., buffer overflow, command injection, information disclosure)
- **Affected file(s):** full path(s) of the file(s) related to the vulnerability
- **Location of the source code:** tag/branch/commit or direct URL
- **Steps to reproduce** the issue
- **Proof-of-concept or exploit code** (if available)
- **Impact** of the issue, including how an attacker might exploit it

This information will help the MSRC team triage your report more quickly.
